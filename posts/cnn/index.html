卷积神经网络

卷积神经网络（convolutional neural network，CNN）是一类强大的、为处理图像数据而设计的神经网络。 基于卷积神经网络架构的模型在计算机视觉领域中已经占主导地位，当今几乎所有的图像识别、目标检测或语义分割相关的学术竞赛和商业应用都以这种方法为基础。

对于图像的分类，我们之前是先用一个展平层nn.Flatten()将图片变成一维向量，再通过全连接和softmax对每一类别的概率进行预测
然而这样存在一些问题，首先把图片展平意味着丢失了图片的空间信息，并且对于尺寸较大的图片来说，需要训练的参数特别多
基于图片中物体的平移不变性，我们提出了卷积神经网络（Convolutional Neural Network, CNN）

1 卷积网络主干#

1.1 卷积层#

卷积神经网络里的卷积含义其实和数学上是不太一样的，数学上做对图像做离散二维卷积的时候，需要把卷积核水平垂直翻转，然后再在图片上滑动按位相乘；
而这里的卷积并不需要对卷积核做翻转，其实二者是等价的，最后训练得到的卷积核在水平和垂直翻转之后和严格定义的卷积核应该是一致的；
我们这边的卷积操作其实应该称作“互相关运算”，但是为了保持术语一致，我们继续把这样的操作叫做卷积。

对图像做卷积的过程其实是在对图像抽取特征，我们手动设置的卷积核如Sobel算子，Canny算子之类的，就可以提取到图像的边缘特征，
而卷积神经网络的卷积核是网络自己学习而来的，通过多个卷积层的堆叠，网络逐步抽取图像的特征，从而对图像进行分类。

1.2 填充和步幅#

填充（padding）就是在输入图像的周围补充元素（通常是0），我们知道对图片运用卷积核卷积后，图片通常会缩小，而为了保持图像前后大小一致，我们就需要使用padding，而padding的大小就是由卷积核大小(kernel_size)决定。
我们一般采用奇数大小的kernel_size，比如1,3,5,7,11之类的，这样做的好处是，我们padding只需要等于(kernel_size-1)/2就可以保证卷积前后图像大小一致

步幅（stride）是每次滑动元素的数量，通常可以决定卷积后图像的大小，比如我们设置stride为2，那图像的长和宽就会缩小一倍，
通常缩小图像的时候我们也会增加通道数来保持特征不丢失。

如果kernel_size不是正方形，padding和stride在高和宽上也会设不同的值，但是实践中很少使用。

1.3 通道#

这里说的通道其实不止是输入图像的RGB三通道这种，更主要的是在卷积的过程中，我们会一步步增加图像的通道数。
刚有提到在一层层卷积的过程中，我们缩小图像的同时也会增加通道，以此来保持图像的特征。

对于N个输入通道，N个输出通道，我们需要几个卷积核？如果只用N个卷积核，那通道间就没有交互，因此网络就无法学习到通道间的关系。
更一般的，在构建卷积神经网络的时候我们可以任意指定有N个输入通道，M个输出通道，对于每个输入通道都有M个卷积核映射到这M个输出通道，所以一共需要NxM个卷积核；这有点类似全连接网络，每一个通道就是一个结点。

1x1卷积看起来没有多大意义，但是它可以把单个像素的所有通道上的信息映射到该像素输出的所有通道上，因此常常用来调整输出的通道数量，适当降低模型的复杂程度。

nn.Conv2d(256, 384, kernel_size=3, padding=1)   # 输入通道数256，输出通道数384

1.4 池化层(汇聚层)#

池化层（pooling层）可以聚合某一层相邻像素之间的信息，因此可以有效减轻卷积层对位置的过度敏感
它可以看作是一个特殊的卷积层，只是我们人为的给他设计了卷积核，并且只对每一通道单独起作用，而不涉及通道间的交互，因此输入和输出的通道数是一样的。 池化层是确定的，没有可学习参数，分为最大池化层（maximum pooling）和平均池化层（average pooling）。

池化层同样可以设置strike和padding，通常我们不设置padding，而使用和池化窗口大小一样的strike来减小图像的高度和宽度。

Adaptive Pooling（自适应池化）是池化层的一个变体，常用在卷积网络的最后阶段，比如 VGG、ResNet、DenseNet 的尾部，用来把任意大小的特征图压缩成固定大小（常见是 1×1）以便接入全连接层。

nn.MaxPool2d(kernel_size=3, stride=2)   # 最大池化
nn.AvgPool2d(kernel_size=2, stride=2)   # 平均池化
nn.AdaptiveAvgPool2d((1, 1))            # 把任意大小的特征图压缩成1*1

1.5 批量规范化#

训练深层的卷积神经网络会遇到因各层数据分布不均而梯度消失，导致底层的参数无法更新，使训练继续不下去。
批量规范化（batch normalization）可以有效解决这个问题
其思想是：在模型训练过程中，用小批量的均值和标准差，不断调整神经网络的中间输出，使整个神经网络各层的中间输出值更加稳定；
在预测的时候则使用保存的均值和方差的移动平均值。

批量规范化在全连接层和卷积层做法不同：

  • 在全连接层中，计算特征维上的均值和方差(dim=0)
  • 在卷积层中，计算通道维上（axis=1）的均值和方差(dim=(0, 2, 3))
nn.BatchNorm2d(6)       # 2d是卷积层，6表示输出通道数
nn.BatchNorm1d(84)      # 1d是全连接层，84表示输出数量

2 经典卷积网络#

卷积神经网络的发展经历了多个阶段，从最初的 LeNet 到后来的 ResNet、DenseNet，结构逐步加深、模块逐步复杂，但核心思想都是在层层卷积中提取图像特征。

2.1 LeNet#

LeNet 是最早的卷积神经网络之一，由 Yann LeCun 提出，用于手写数字识别（MNIST 数据集）。
网络结构：

net = nn.Sequential(
    nn.Conv2d(1, 6, kernel_size=5, padding=2), nn.Sigmoid(),
    nn.AvgPool2d(kernel_size=2, stride=2),
    nn.Conv2d(6, 16, kernel_size=5), nn.Sigmoid(),
    nn.AvgPool2d(kernel_size=2, stride=2),
    nn.Flatten(),
    nn.Linear(16 * 5 * 5, 120), nn.Sigmoid(),
    nn.Linear(120, 84), nn.Sigmoid(),
    nn.Linear(84, 10))

2.2 AlexNet#

AlexNet 是深度学习在图像识别上的突破性成果，它在 ImageNet 竞赛中以巨大优势夺冠，标志着深度学习时代的到来。
网络结构：

net = nn.Sequential(
    # 这里使用一个11*11的更大窗口来捕捉对象。
    # 同时，步幅为4，以减少输出的高度和宽度。
    # 另外，输出通道的数目远大于LeNet
    nn.Conv2d(1, 96, kernel_size=11, stride=4, padding=1), nn.ReLU(),
    nn.MaxPool2d(kernel_size=3, stride=2),
    # 减小卷积窗口，使用填充为2来使得输入与输出的高和宽一致，且增大输出通道数
    nn.Conv2d(96, 256, kernel_size=5, padding=2), nn.ReLU(),
    nn.MaxPool2d(kernel_size=3, stride=2),
    # 使用三个连续的卷积层和较小的卷积窗口。
    # 除了最后的卷积层，输出通道的数量进一步增加。
    # 在前两个卷积层之后，汇聚层不用于减少输入的高度和宽度
    nn.Conv2d(256, 384, kernel_size=3, padding=1), nn.ReLU(),
    nn.Conv2d(384, 384, kernel_size=3, padding=1), nn.ReLU(),
    nn.Conv2d(384, 256, kernel_size=3, padding=1), nn.ReLU(),
    nn.MaxPool2d(kernel_size=3, stride=2),
    nn.Flatten(),
    # 这里，全连接层的输出数量是LeNet中的好几倍。使用dropout层来减轻过拟合
    nn.Linear(6400, 4096), nn.ReLU(),
    nn.Dropout(p=0.5),
    nn.Linear(4096, 4096), nn.ReLU(),
    nn.Dropout(p=0.5),
    nn.Linear(4096, 1000))

2.3 VGG#

VGG网络的核心思想是用小卷积核堆叠代替大卷积核，用深度替代广度。
模型结构：

经典卷积神经网络的基本组成部分是下面的这个序列：

  1. 带填充以保持分辨率的卷积层；
  2. 非线性激活函数，如ReLU；
  3. 汇聚层，如最大汇聚层。

一个VGG块与之类似，先是一系列3×3的卷积，然后ReLU，最后再做MaxPooling来使分辨率减半。

def vgg_block(num_convs, in_channels, out_channels):
    layers = []
    for _ in range(num_convs):
        layers.append(nn.Conv2d(in_channels, out_channels,
                                kernel_size=3, padding=1))
        layers.append(nn.ReLU())
        in_channels = out_channels
    layers.append(nn.MaxPool2d(kernel_size=2,stride=2))
    return nn.Sequential(*layers)

原始VGG网络有5个卷积块，其中前两个块各有一个卷积层，后三个块各包含两个卷积层。 第一个模块有64个输出通道，每个后续模块将输出通道数量翻倍，直到该数字达到512。由于该网络使用8个卷积层和3个全连接层，因此它通常被称为VGG-11。
VGG的代表模型是VGG-16和VGG-19。

# VGG-11
def vgg(conv_arch):
    conv_blks = []
    in_channels = 1
    # 卷积层部分
    for (num_convs, out_channels) in conv_arch:
        conv_blks.append(vgg_block(num_convs, in_channels, out_channels))
        in_channels = out_channels

    return nn.Sequential(
        *conv_blks, nn.Flatten(),
        # 全连接层部分
        nn.Linear(out_channels * 7 * 7, 4096), nn.ReLU(), nn.Dropout(0.5),
        nn.Linear(4096, 4096), nn.ReLU(), nn.Dropout(0.5),
        nn.Linear(4096, 10))

conv_arch = ((1, 64), (1, 128), (2, 256), (2, 512), (2, 512))
# 卷积部分： 卷积1次，64通道 -> 卷积1次，128通道 -> 卷积2次，256通道 -> 卷积2次，512通道 -> 卷积2次，512通道
# 全连接部分： 512*7*7 -> 4096 -> 4096 -> 10
net = vgg(conv_arch)

2.4 NiN#

全连接的参数量是非常巨大的，NiN 用 1×1 卷积核代替传统的全连接层，它显著减少了模型所需参数的数量。
然而，在实践中，这种设计有时会增加训练模型的时间。
模型结构：

一个NiN块由一次卷积和两次1×1卷积组成

def nin_block(in_channels, out_channels, kernel_size, strides, padding):
    return nn.Sequential(
        nn.Conv2d(in_channels, out_channels, kernel_size, strides, padding),
        nn.ReLU(),
        nn.Conv2d(out_channels, out_channels, kernel_size=1), nn.ReLU(),
        nn.Conv2d(out_channels, out_channels, kernel_size=1), nn.ReLU())

使用两个 1×1 卷积可能是因为：

  • 第一个 1×1 卷积：在通道上混合信息（增加特征维度非线性）
  • 第二个 1×1 卷积：进一步非线性映射或调整通道（压缩/整合特征）

实际上在小数据集改成一次卷积没有太大性能变化

NiN使用窗口形状为$11\times 11$、$5\times 5$和$3\times 3$的卷积层，输出通道数量与AlexNet中的相同。 每个NiN块后有一个最大汇聚层，汇聚窗口形状为$3\times 3$，步幅为2。 NiN使用一个NiN块，其输出通道数等于标签类别的数量。最后放一个全局平均汇聚层（global average pooling layer），生成一个对数几率 （logits）。

net = nn.Sequential(
    nin_block(1, 96, kernel_size=11, strides=4, padding=0),
    nn.MaxPool2d(3, stride=2),
    nin_block(96, 256, kernel_size=5, strides=1, padding=2),
    nn.MaxPool2d(3, stride=2),
    nin_block(256, 384, kernel_size=3, strides=1, padding=1),
    nn.MaxPool2d(3, stride=2),
    nn.Dropout(0.5),
    # 标签类别数是10
    nin_block(384, 10, kernel_size=3, strides=1, padding=1),
    nn.AdaptiveAvgPool2d((1, 1)),
    # 将四维的输出转成二维的输出，其形状为(批量大小,10)
    nn.Flatten())

2.5 GoogLeNet#

核心是 Inception 模块，通过并行使用不同尺寸的卷积核，提升特征提取的多样性。

Inception块由四条并行路径组成。
前三条路径使用窗口大小为$1\times 1$、$3\times 3$和$5\times 5$的卷积层，从不同空间大小中提取信息。
中间的两条路径在输入上执行$1\times 1$卷积，以减少通道数，从而降低模型的复杂性。
第四条路径使用$3\times 3$最大汇聚层，然后使用$1\times 1$卷积层来改变通道数。
这四条路径都使用合适的填充来使输入与输出的高和宽一致，最后我们将每条线路的输出在通道维度上连结，并构成Inception块的输出。在Inception块中，通常调整的超参数是每层输出通道数。

class Inception(nn.Module):
    # c1--c4是每条路径的输出通道数
    def __init__(self, in_channels, c1, c2, c3, c4, **kwargs):
        super(Inception, self).__init__(**kwargs)
        # 线路1，单1x1卷积层
        self.p1_1 = nn.Conv2d(in_channels, c1, kernel_size=1)
        # 线路2，1x1卷积层后接3x3卷积层
        self.p2_1 = nn.Conv2d(in_channels, c2[0], kernel_size=1)
        self.p2_2 = nn.Conv2d(c2[0], c2[1], kernel_size=3, padding=1)
        # 线路3，1x1卷积层后接5x5卷积层
        self.p3_1 = nn.Conv2d(in_channels, c3[0], kernel_size=1)
        self.p3_2 = nn.Conv2d(c3[0], c3[1], kernel_size=5, padding=2)
        # 线路4，3x3最大汇聚层后接1x1卷积层
        self.p4_1 = nn.MaxPool2d(kernel_size=3, stride=1, padding=1)
        self.p4_2 = nn.Conv2d(in_channels, c4, kernel_size=1)

    def forward(self, x):
        p1 = F.relu(self.p1_1(x))
        p2 = F.relu(self.p2_2(F.relu(self.p2_1(x))))
        p3 = F.relu(self.p3_2(F.relu(self.p3_1(x))))
        p4 = F.relu(self.p4_2(self.p4_1(x)))
        # 在通道维度上连结输出
        return torch.cat((p1, p2, p3, p4), dim=1)

GoogLeNet一共使用9个Inception块和全局平均汇聚层的堆叠来生成其估计值。Inception块之间的最大汇聚层可降低维度。
模型结构：

b1 = nn.Sequential(nn.Conv2d(1, 64, kernel_size=7, stride=2, padding=3),
                   nn.ReLU(),
                   nn.MaxPool2d(kernel_size=3, stride=2, padding=1))
b2 = nn.Sequential(nn.Conv2d(64, 64, kernel_size=1),
                   nn.ReLU(),
                   nn.Conv2d(64, 192, kernel_size=3, padding=1),
                   nn.ReLU(),
                   nn.MaxPool2d(kernel_size=3, stride=2, padding=1))
b3 = nn.Sequential(Inception(192, 64, (96, 128), (16, 32), 32),
                   Inception(256, 128, (128, 192), (32, 96), 64),
                   nn.MaxPool2d(kernel_size=3, stride=2, padding=1))
b4 = nn.Sequential(Inception(480, 192, (96, 208), (16, 48), 64),
                   Inception(512, 160, (112, 224), (24, 64), 64),
                   Inception(512, 128, (128, 256), (24, 64), 64),
                   Inception(512, 112, (144, 288), (32, 64), 64),
                   Inception(528, 256, (160, 320), (32, 128), 128),
                   nn.MaxPool2d(kernel_size=3, stride=2, padding=1))
b5 = nn.Sequential(Inception(832, 256, (160, 320), (32, 128), 128),
                   Inception(832, 384, (192, 384), (48, 128), 128),
                   nn.AdaptiveAvgPool2d((1,1)),
                   nn.Flatten())

net = nn.Sequential(b1, b2, b3, b4, b5, nn.Linear(1024, 10))

感觉Inception块的参数都是试出来的….

2.6 ResNet#

唯一真神，允许我们训练特别深的网络
残差网络的核心思想是：每个附加层都应该更容易地包含原始函数作为其元素之一。

我们把原来网络要学的x到f(x)的映射，变成了只需要学x到f(x)-x的映射，这样就很容易捕捉映射的细微波动，
不仅如此，原来网络一加深，如果在某一层参数很小，会导致梯度消失；
而残差块(residual block)则可以通过原来的x那条路径继续传播，因此我们的网络可以训练的特别深。
进一步改进是在x那条路径上添加1×1卷积层，来调整通道数；在卷积层后添加Batch Normalization。

class Residual(nn.Module):
    def __init__(self, input_channels, num_channels,
                 use_1x1conv=False, strides=1):
        super().__init__()
        self.conv1 = nn.Conv2d(input_channels, num_channels,
                               kernel_size=3, padding=1, stride=strides)
        self.conv2 = nn.Conv2d(num_channels, num_channels,
                               kernel_size=3, padding=1)
        if use_1x1conv:
            self.conv3 = nn.Conv2d(input_channels, num_channels,
                                   kernel_size=1, stride=strides)
        else:
            self.conv3 = None
        self.bn1 = nn.BatchNorm2d(num_channels)
        self.bn2 = nn.BatchNorm2d(num_channels)

    def forward(self, X):
        Y = F.relu(self.bn1(self.conv1(X)))
        Y = self.bn2(self.conv2(Y))
        if self.conv3:
            X = self.conv3(X)
            Y += X
        return F.relu(Y)

每个模块有4个卷积层（不包括恒等映射的1×1卷积层）。 加上第一个7×7卷积层和最后一个全连接层，共有18层。 因此，这种模型通常被称为ResNet-18。
通过配置不同的通道数和模块里的残差块数可以得到不同的ResNet模型，例如更深的含152层的ResNet-152。
模型结构：

b1 = nn.Sequential(nn.Conv2d(1, 64, kernel_size=7, stride=2, padding=3),
                   nn.BatchNorm2d(64), nn.ReLU(),
                   nn.MaxPool2d(kernel_size=3, stride=2, padding=1))
                   
def resnet_block(input_channels, num_channels, num_residuals,
                 first_block=False):
    blk = []
    for i in range(num_residuals):
        if i == 0 and not first_block:
            blk.append(Residual(input_channels, num_channels,
                                use_1x1conv=True, strides=2))
        else:
            blk.append(Residual(num_channels, num_channels))
    return blk

b2 = nn.Sequential(*resnet_block(64, 64, 2, first_block=True))
b3 = nn.Sequential(*resnet_block(64, 128, 2))
b4 = nn.Sequential(*resnet_block(128, 256, 2))
b5 = nn.Sequential(*resnet_block(256, 512, 2))
net = nn.Sequential(b1, b2, b3, b4, b5,
                    nn.AdaptiveAvgPool2d((1,1)),
                    nn.Flatten(), nn.Linear(512, 10))

2.7 DenseNet#

是ResNet思想的进一步发展，DenseNet的一个Dense Block中每层都与之前的所有层直接连接
与ResNet不同的是，在跨层连接时把相加换成了连结。

def conv_block(input_channels, num_channels):
    return nn.Sequential(
        nn.BatchNorm2d(input_channels), nn.ReLU(),
        nn.Conv2d(input_channels, num_channels, kernel_size=3, padding=1))

class DenseBlock(nn.Module):
    def __init__(self, num_convs, input_channels, num_channels):
        super(DenseBlock, self).__init__()
        layer = []
        for i in range(num_convs):
            layer.append(conv_block(
                num_channels * i + input_channels, num_channels))
        self.net = nn.Sequential(*layer)

    def forward(self, X):
        for blk in self.net:
            Y = blk(X)
            # 连接通道维度上每个块的输入和输出
            X = torch.cat((X, Y), dim=1)
        return X

由于每个稠密块都会带来通道数的增加，使用过多则会过于复杂化模型。 而过渡层可以用来控制模型复杂度。 它通过1×1卷积层来减小通道数，并使用步幅为2的平均汇聚层减半高和宽，从而进一步降低模型复杂度。

def transition_block(input_channels, num_channels):
    return nn.Sequential(
        nn.BatchNorm2d(input_channels), nn.ReLU(),
        nn.Conv2d(input_channels, num_channels, kernel_size=1),
        nn.AvgPool2d(kernel_size=2, stride=2))

模型结构：

b1 = nn.Sequential(
    nn.Conv2d(1, 64, kernel_size=7, stride=2, padding=3),
    nn.BatchNorm2d(64), nn.ReLU(),
    nn.MaxPool2d(kernel_size=3, stride=2, padding=1))

# num_channels为当前的通道数
num_channels, growth_rate = 64, 32
num_convs_in_dense_blocks = [4, 4, 4, 4]
blks = []
for i, num_convs in enumerate(num_convs_in_dense_blocks):
    blks.append(DenseBlock(num_convs, num_channels, growth_rate))
    # 上一个稠密块的输出通道数
    num_channels += num_convs * growth_rate
    # 在稠密块之间添加一个转换层，使通道数量减半
    if i != len(num_convs_in_dense_blocks) - 1:
        blks.append(transition_block(num_channels, num_channels // 2))
        num_channels = num_channels // 2

net = nn.Sequential(
    b1, *blks,
    nn.BatchNorm2d(num_channels), nn.ReLU(),
    nn.AdaptiveAvgPool2d((1, 1)),
    nn.Flatten(),
    nn.Linear(num_channels, 10))

3 轻量化网络#

随着深度学习在移动端与嵌入式设备上的应用增多，传统的卷积神经网络（如 VGG、ResNet）计算量和参数量过大，不适合实时或低功耗场景。
深度可分离卷积（Depthwise Separable Convolution）就是轻量化网络的关键构件之一。
它将传统卷积分成两步：

  1. Depthwise Conv：每个输入通道单独卷积（提取空间特征）；
  2. Pointwise Conv (1×1)：跨通道线性组合（提取通道特征）。 输出有几个通道就用几个1×1的卷积核

MobileNetV1就是将常规卷积替换为深度可分离卷积的VGG网络，对于 3×3 卷积，计算量可减少约 8~9倍。

还有一个常见的网络是ShuffleNet，它采用分组卷积 + 通道混洗

  1. Group Convolution（分组卷积）：把通道分成若干组，每组单独卷积，降低计算量；
  2. Channel Shuffle（通道混洗）：不同组之间的信息无法直接交互，因此引入“混洗”操作
    • 类似洗牌，把各组通道交叉打乱；
    • 促进跨组信息流动。
